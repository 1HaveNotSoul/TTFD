ЧЕКЛИСТ: ПОЧЕМУ POSTGRESQL НЕ РАБОТАЕТ
═══════════════════════════════════════════════════════════════

Проверь каждый пункт:

□ 1. PostgreSQL база создана на Railway
   - Railway → должен быть сервис "Postgres"
   - Статус: Online

□ 2. DATABASE_URL добавлен в бота
   - Railway → TTFD (бот) → Variables
   - Должна быть переменная DATABASE_URL
   - Значение начинается с postgresql://

□ 3. DATABASE_URL правильный
   - Скопирован из Postgres → Variables → DATABASE_URL
   - Не изменён вручную
   - Полный URL (не обрезан)

□ 4. Бот перезапущен после добавления DATABASE_URL
   - Railway → TTFD → Settings → Restart
   - ИЛИ новый деплой после git push

□ 5. requirements.txt содержит psycopg2
   - Открой discord-bot/requirements.txt
   - Должна быть строка: psycopg2-binary==2.9.9

□ 6. Код бота обновлён
   - Последний git push был сделан
   - Railway задеплоил новую версию

═══════════════════════════════════════════════════════════════
ЕСЛИ ВСЁ ОТМЕЧЕНО, НО НЕ РАБОТАЕТ:
═══════════════════════════════════════════════════════════════

Покажи мне:

1. Скриншот Railway → TTFD → Variables
   (закрой значения, покажи только названия)

2. Логи бота при запуске:
   Railway → TTFD → Deployments → View Logs
   (первые 50 строк)

3. Результат команд в Discord:
   !work
   !balance
   (перезапусти бота)
   !balance

ЧАСТЫЕ ОШИБКИ:
═══════════════════════════════════════════════════════════════

❌ DATABASE_URL добавлен в Postgres, а не в TTFD
   → Добавь в TTFD (бот), не в Postgres

❌ DATABASE_URL не полный (обрезан)
   → Скопируй заново, весь URL

❌ Бот не перезапущен после добавления переменной
   → Перезапусти: Settings → Restart

❌ Старая версия кода без поддержки PostgreSQL
   → git push последних изменений

❌ psycopg2 не установлен
   → Проверь requirements.txt
